## 2. 管理摘要

用 3-5 句话说明整体判断：

- 需求流入：新增需求 __ 条，较上周期增加 / 减少 __ 条。
- 需求吞吐：完成需求 __ 条，净增需求 __ 条。
- 重点需求：P0/P1 需求 __ 条，其中存在风险 __ 条。
- 主要风险：延期、今日到期、排期信息不全、资源过载、评审/排期阶段堆积。
- PMO 结论：整体可控 / 局部风险 / 明显失控风险。

## 6. P0/P1 高优需求治理

责任人口径：

- 节点风险责任人优先取延期、今日到期、排期信息不全等风险节点的负责人或子任务负责人。
- 需求当前负责人仅作为辅助字段；当无法获取节点负责人时，才回退使用，并需标注为“需求当前负责人”。

| 工作项ID | 需求名称 | 优先级 | 当前状态 | 风险节点/风险类型 | 风险节点负责人 | 需求当前负责人 | 是否排期 | 计划完成时间 | PMO 动作 |
|---|---|---|---|---|---|---|---|---|---|
|  |  |  |  |  |  |  |  |  |  |

重点关注：

- 是否有明确负责人。
- 是否有完整排期和估分。
- 是否卡在评审、排期、测试、上线等阶段。
- 是否存在延期、今日到期或跨团队依赖未闭环。

PMO 介入规则：

- P0/P1 需求无排期或无负责人。

- 关键节点今日到期但未完成。

- 资源明显过载导致承诺不可达。
